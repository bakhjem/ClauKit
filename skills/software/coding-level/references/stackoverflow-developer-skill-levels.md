# Stack Overflow · Measurable Skill Levels for Developers

**Source**: https://stackoverflow.blog/2022/07/28/measurable-and-meaningful-skill-levels-for-developers/

## The 5-level framework

Stack Overflow proposes a model based on **observable behavior**, not years of experience:

| Level | Name | Observable trait |
|---|---|---|
| 1 | Novice | Follows recipes; struggles when example doesn't match |
| 2 | Advanced beginner | Recognizes patterns; can adapt examples |
| 3 | Competent | Plans and executes routine tasks independently |
| 4 | Proficient | Debugs unfamiliar code; sees systems, not lines |
| 5 | Expert | Sets standards; mentors; recognizes when rules should bend |

## Why behavior > years

Years of experience are a poor proxy. A 10-year developer who's done the same CRUD job all decade may sit at level 3; a 2-year prodigy who's shipped a compiler may be at 4.

## Activity domains

Stack Overflow stresses skill is **domain-specific**. Map levels per activity:

- Syntax fluency
- Debugging complex code
- System / architecture design
- Refactoring legacy code
- Code review and mentorship

A developer can be a 5 in API design and a 2 in ML — both are real coding skills.

## How coding-level skill uses this

`coding-level` collapses this richness into a single 0–5 scalar for output-tuning purposes — a pragmatic simplification, not an HR framework. For a real assessment, use the full multi-domain map.

## Why authoritative

Stack Overflow surveys millions of developers annually; their skill model is grounded in observed self-report and hiring practice.
