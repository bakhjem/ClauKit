# Agile Retrospective Methods & Timing

**Source**: https://www.theprojectgroup.com/blog/en/agile-retrospective/ (2024)

## Format catalog

| Format | Best when |
|---|---|
| Start / Stop / Continue | Team needs an action bias; great for new teams. |
| Glad / Sad / Mad | Emotional residue is present; helps surface morale issues. |
| 4 L's (Liked / Learned / Lacked / Longed-for) | Mature teams; richer than 3-bucket formats. |
| Sailboat | Visual; useful for roadmap-shaped retros (wind = helps, anchors = slows, rocks = risks, island = goal). |
| 5 Whys | Single-incident root cause; drives to systemic fix. |
| Mad / Sad / Glad followed by dot-voting | Prioritizes which themes to act on when many surface. |

## Timing guidance

- **Weekly**: 20–30 min. Use lightweight formats (Start/Stop/Continue).
- **Sprint (2-week)**: 45–60 min. Any format.
- **Quarterly**: 90 min, often as workshop. Sailboat or 4 L's suit longer horizons.
- **Post-incident**: open-ended; use 5 Whys + blameless framing.

## Critical rule

The retrospective is **wasted** without measurable, owned action items. The guide stresses that the value of a retro is the **delta in behavior** between this sprint and the next — not the discussion itself.
