### Finding H-1 — BROKEN-ACCESS-CONTROL

**File:** `.claude/skills/software/payment-integration/scripts/sepay-webhook-verify.js`  
**Lines:** 20, 159

**Description:**  
Constructor defaults `authType = 'none'`. When env var `SEPAY_WEBHOOK_AUTH_TYPE` is unset (which is the case in default deployments), all webhook calls bypass authentication entirely. Any attacker who knows the webhook URL can inject arbitrary payment transactions.

**Impact:**  
Fake payment confirmations injected without any credential. Direct financial fraud risk in production.

**Data flow:**  
L1 (external webhook caller) → `process()` → business logic — **no L1→L4 gate**.

**Code (before):**
```js
// sepay-webhook-verify.js:20
constructor(authType = 'none', apiKey = null) {

// sepay-webhook-verify.js:159
const authType = process.env.SEPAY_WEBHOOK_AUTH_TYPE || 'none';
```

**Code (after):**
```js
constructor(authType, apiKey = null) {
  if (!authType) throw new Error('authType is required. Set SEPAY_WEBHOOK_AUTH_TYPE env var.');
  this.authType = authType;

// sepay-webhook-verify.js:159
const authType = process.env.SEPAY_WEBHOOK_AUTH_TYPE;
if (!authType) {
  console.error('❌ SEPAY_WEBHOOK_AUTH_TYPE must be set (api_key or oauth2)');
  process.exit(1);
}
```

**References:** OWASP A01:2021, CWE-284

### Finding H-2 — JWT-NONE-ALGORITHM (OAuth2 token presence-only check)

**File:** `.claude/skills/software/payment-integration/scripts/sepay-webhook-verify.js`  
**Lines:** 49–58

**Description:**  
OAuth2 path checks only that `Authorization` header starts with `Bearer ` — it does NOT verify the token's signature, expiry, issuer, or audience with the OAuth2 provider. Any string starting with `Bearer ` is accepted.

**Impact:**  
Attacker sends `Authorization: Bearer anything` and gains authenticated access to webhook processing.

**Data flow:**  
L1 (request header, attacker-controlled) → presence check only → accepted as authenticated.

**Code (before):**
```js
if (!authHeader || !authHeader.startsWith('Bearer ')) {
  throw new Error('Missing or invalid OAuth2 Bearer token');
}
// In production, verify token with OAuth2 provider
console.log('✓ OAuth2 token present (full verification needed in production)');
return true;
```

**Code (after):**
```js
if (!authHeader || !authHeader.startsWith('Bearer ')) {
  throw new Error('Missing or invalid OAuth2 Bearer token');
}
const token = authHeader.slice(7);
// Verify token with OAuth2 introspection endpoint
const isValid = await this.introspectToken(token); // must be implemented
if (!isValid) throw new Error('OAuth2 token validation failed');
return true;
```

**References:** OWASP A07:2021, RFC 7662 (Token Introspection), CWE-287
