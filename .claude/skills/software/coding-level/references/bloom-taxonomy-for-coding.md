# Bloom's Taxonomy Applied to Coding

**Source**: https://en.wikipedia.org/wiki/Bloom%27s_taxonomy

## Original taxonomy

A 6-level model of cognitive complexity, originally for education:

1. **Remember** — recall facts, terms.
2. **Understand** — explain ideas in own words.
3. **Apply** — use info in a new situation.
4. **Analyze** — break information into parts; see relationships.
5. **Evaluate** — judge based on criteria.
6. **Create** — produce new or original work.

## Mapped to coding

| Bloom level | Coding behavior | coding-level mapping |
|---|---|---|
| Remember | Recall syntax, find a method in docs | 0–1 |
| Understand | Read code and predict behavior | 1–2 |
| Apply | Modify existing code to solve a new variant | 2–3 |
| Analyze | Trace bugs through unfamiliar systems | 3–4 |
| Evaluate | Choose between architectures with reasoning | 4–5 |
| Create | Design novel solutions, new abstractions | 5 |

## Why this fits output tuning

A developer at the Apply level wants worked examples they can adapt. A developer at Evaluate wants criteria for comparison, not examples. The same "right answer" delivered at the wrong cognitive level lands wrong.

## Caveats

- A developer is not at one Bloom level across all domains.
- High levels do not imply low levels are mastered; experts sometimes forget basics they no longer use.
- The taxonomy is a rough scale, not a measurement instrument.

## Why authoritative

Bloom's Taxonomy (revised 2001) is the most-cited cognitive framework in education research, with decades of applied use in CS pedagogy.
