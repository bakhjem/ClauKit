# Aha! · Agile Retrospective Guide

**Source**: https://www.aha.io/roadmapping/guide/agile/what-is-an-agile-retrospective

## Continuous improvement loop

A retrospective is one step of a four-step loop: **Reflect → Decide → Commit → Measure**. Without the measure step, retros become theater.

- **Reflect**: gather data (silent writing, sticky notes, async docs). Surface observations, not solutions.
- **Decide**: cluster, vote, pick 1–3 high-impact themes. Resist tackling everything.
- **Commit**: action items with owner + due date + measurable outcome. "Improve communication" is not an action item; "schedule a weekly 15-min sync, owner @alice, starting Monday" is.
- **Measure**: at the next retro, review previous actions. Mark done, in-progress, abandoned. Abandoned actions are signal — either the theme wasn't important or the action was wrong.

## Psychological safety

The retro fails if people don't speak. Three practices Aha! recommends:
1. **Silent writing first** to neutralize loudest-voice dominance.
2. **Rotate facilitator** so no one is permanently "in charge."
3. **Blameless framing**: "what process let this happen?" not "who did this?"

## Quarterly review

In addition to per-sprint retros, run a **quarterly meta-retro** on the retros themselves. Are themes recurring? Are action items closing? This catches drift the weekly cadence misses.
